# Plan de séquence - Semaine 01

## Titre
Introduction à l'IdO et architecture: Raspberry Pi 5 serveur

## Objectifs de la semaine
- Comprendre les concepts fondamentaux de l'Internet des Objets (IdO)
- Configurer un Raspberry Pi 5 comme serveur IdO
- Installer et configurer un courtier MQTT Mosquitto
- Exposer le serveur de manière sécurisée via Cloudflare Tunnel
- Prendre en main la carte LilyGO A7670G (ESP32 + LTE + GPS)
- Configurer l'environnement de développement Arduino CLI
- Acquérir les bases de Python pour l'interface tactile

## Contenu théorique

### Introduction à l'Internet des Objets
- Définition et concepts clés de l'IdO
- Architectures des systèmes IdO (edge, fog, cloud)
- Composants matériels typiques (capteurs, actionneurs, microcontrôleurs)
- Protocoles de communication courants (MQTT, HTTP, CoAP)
- Enjeux de sécurité dans l'IdO

### Le Raspberry Pi 5 comme serveur IdO
- Caractéristiques techniques du Raspberry Pi 5
- Avantages pour les applications IdO
- Configuration initiale et sécurisation
- Introduction à Cloudflare Tunnel pour l'exposition sécurisée

### Présentation du LilyGO A7670G
- Architecture ESP32 avec module LTE et GPS
- Cas d'utilisation en IdO
- Environnement de développement Arduino CLI

## Activités

### Théorie (2h)
- Présentation des concepts IdO et des architectures
- Démonstration de l'infrastructure cible du cours
- Introduction aux outils et plateformes utilisés

### Laboratoire (3h)
- Configuration du Raspberry Pi 5 comme serveur
- Installation du courtier MQTT Mosquitto
- Configuration de Cloudflare Tunnel
- Prise en main initiale de la carte LilyGO A7670G
- Configuration de l'environnement Arduino CLI
- Premiers pas avec Python pour l'interface tactile

## Travaux hors classe
- Installation des outils de développement
- Création des comptes nécessaires (Cloudflare, etc.)
- Lecture sur les architectures IdO et la sécurité d'exposition
- Exercices Python de base
