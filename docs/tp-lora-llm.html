TP Intégration LoRa + LLM

Pondération : 20% Remise : semaine 12 Capacité 1 : 15% + Capacité 2 : 5%

Travail pratique évalué réalisé aux semaines 11 et 12. Il consiste à construire une liaison LoRa point à point entre deux T-Beam Supreme, puis à enrichir les données reçues par un appel à un LLM (Groq) avant de publier le résultat sur le broker MQTT.

Objectifs

  • Mettre en œuvre une communication LoRa sans Meshtastic avec la librairie RadioLib
  • Concevoir un pipeline capteur → LoRa → WiFi → LLM → MQTT → actionneur
  • Intégrer un LLM dans une boucle de décision IoT embarquée
  • Gérer les secrets (clés API, identifiants WiFi) via config.h

Architecture

Deux T-Beam Supreme communiquent en LoRa 915 MHz :

Émetteur — lit un potentiomètre et transmet la valeur via LoRa (JSON).
Récepteur — reçoit la trame LoRa, mesure RSSI/SNR.
WiFi + LLM — le récepteur appelle l'API Groq (llama-3.3-70b-versatile) pour analyser les données.
MQTT — le résultat structuré (status, action) est publié sur le broker Mosquitto.
Actionneur — une DEL réagit selon la décision du LLM (on / off).

Matériel requis

  • 2 × LilyGO T-Beam Supreme (ESP32-S3 + SX1262)
  • 1 × Potentiomètre 10 kΩ
  • 2 × DEL (status émetteur et réaction récepteur)
  • Câbles USB-C, fils de connexion, breadboard
  • Accès à un broker MQTT (Mosquitto) et une clé API Groq

Étapes suggérées

Semaine 11 — Liaison LoRa et premier appel LLM

  • Initialiser RadioLib (SX1262) sur les deux T-Beam avec la même configuration radio
  • Coder l'émetteur : lecture potentiomètre, envoi JSON via LoRa, DEL status
  • Coder le récepteur : réception LoRa, affichage RSSI/SNR
  • Connexion WiFi sur le récepteur, premier appel à l'API Groq
  • Contrôle de la DEL selon la réponse du LLM

Semaine 12 — Intégration MQTT, fiabilité et remise

  • Publication MQTT du résultat de l'analyse LLM
  • Gestion robuste des erreurs (retry, timeout, reconnexion)
  • Rédaction du README.md et nettoyage du dépôt
  • Vérification finale : aucun secret dans le code ou l'historique git
Sécurité : une clé API présente dans le code source ou l'historique git entraîne une pénalité de -20 points. Utilisez systématiquement config.h (ignoré par git) et fournissez un config.example.h.
