Introduction aux grands modèles de langage (LLM) sur ESP32 et assemblage des PCB reçus du fabricant : appel API HTTP depuis un microcontrôleur, puis soudure du shield du projet de mi-session.

Introduction aux LLM :

  • API compatible OpenAI — requête POST HTTPS avec messages JSON
  • Groq — API gratuite pour accéder à des modèles rapides (Llama)
  • Scénario créatif — le potentiomètre (0-4095) influence la réponse du LLM
  • Sécurité — fichier config.h ignoré par Git, config.example.h commité

Soudure du PCB :

  • Composants du shield — résistances 330 Ω, headers LilyGO A7670E, boutons, LEDs, potentiomètres, MPU6050
  • Vérification — inspection visuelle et test de continuité au multimètre
  • Test fonctionnel — validation avec le firmware du projet de mi-session

Labo : créer un dépôt Git structuré, effectuer un appel API LLM depuis le T-Beam Supreme avec un scénario créatif, puis souder et tester le shield PCB avec le LilyGO A7670E.
