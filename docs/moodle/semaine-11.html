Début du TP évalué (20%) : liaison LoRa point à point codée dans l'IDE Arduino avec analyse LLM et contrôle d'une DEL.

Émetteur LoRa :

  • RadioLib — librairie pour contrôler directement le SX1262 du T-Beam
  • Potentiomètre — lecture ADC (0-4095) envoyée via LoRa
  • OLED — affiche la trame envoyée et la décision reçue
  • DEL action — allumée/éteinte selon la décision du LLM (retour LoRa)
  • Format JSON — message structuré avec ArduinoJson

Récepteur LoRa + WiFi + LLM :

  • Réception LoRa — avec métriques RSSI et SNR
  • OLED — affiche message reçu, RSSI/SNR et réponse du LLM
  • DEL status — clignote à chaque réception et appel LLM
  • Connexion WiFi — pour accéder à l'API LLM
  • Appel LLM — analyse avec structured output obligatoire (schéma JSON défini par l'étudiant)
  • Retour LoRa — renvoie la décision à l'émetteur (liaison bidirectionnelle)

Sécurité :

  • config.h — secrets (WiFi, API key), jamais commité
  • config.example.h — template sans valeurs, commité
  • Pénalités — clé API dans le code = -20 points

Labo : coder l'émetteur (potentiomètre → LoRa, DEL action, OLED) et le récepteur (LoRa → WiFi → LLM → retour LoRa, DEL status, OLED) dans l'IDE Arduino.
