Projet final — Hydro-Limoilou Télémétrie RF

Pondération : 35% Semaines 13 à 15 Capacité 1 : 10% + Capacité 2 : 25%

Le projet final simule le déploiement de 8 sites RF d'un distributeur d'énergie fictif, Hydro-Limoilou. Chaque site héberge un répéteur RF (configuré par l'étudiant·e dans un cours connexe), et le présent projet déploie l'infrastructure de télémétrie qui en surveille les conditions opérationnelles. Un serveur central VM simulant un NOC agrège les 8 sites.

1. Contexte et articulation avec le cours connexe

Chaque étudiant·e configure son propre répéteur RF dans un cours parallèle. Le présent projet en assure le monitoring environnemental et opérationnel :

  • la télémétrie observe les conditions qui peuvent affecter le bon fonctionnement du répéteur (température dans le shelter, vibration du mât d'antenne, intégrité de l'alimentation, intrusion physique)
  • les alarmes signalent les situations qui menacent la disponibilité du répéteur (basculement, inondation, ouverture du cabinet, batterie faible, niveau carburant bas)
  • la convergence vers le serveur central simule un NOC (Network Operations Center) supervisant l'ensemble des 8 répéteurs déployés sur le territoire

2. Architecture globale

Voie LoRa (sites #1-4) — Capteurs → T-Beam SUPREME distant → LoRa mesh → T-Beam SUPREME gateway WiFi → Mosquitto local sur Pi 5 → Cloudflare Tunnel → VM
Voie LTE (sites #5-8) — Capteurs → LilyGO A7670G → MQTT/WSS direct via réseau cellulaire → Cloudflare → Mosquitto local sur Pi 5 → Cloudflare Tunnel → VM
Serveur central VM — souscrit aux 8 brokers, agrège les données, dashboard global de supervision

Chaque site reçoit une combinaison unique de modules breakout (avec headers) à intégrer sur un shield PCB conçu en KiCad. Les shields se montent directement sur l'hôte (T-Beam SUPREME ou LilyGO A7670G).

6. Convention de topics MQTT

Tous les topics sont préfixés par hydro-limoilou/{site-id}/. Site-id = poste-01 à poste-08.

hydro-limoilou/{site-id}/telemetry/{capteur}     # Mesures périodiques
hydro-limoilou/{site-id}/status                  # État périodique du nœud
hydro-limoilou/{site-id}/alarm/{type}            # Évènements ponctuels
hydro-limoilou/{site-id}/actuators/{nom}         # Commandes descendantes

Exemples de payloads

// hydro-limoilou/poste-01/telemetry/temperature
{"value": 22.4, "unit": "C", "ts": 1739500000}

// hydro-limoilou/poste-01/telemetry/vibration
{"x": 0.02, "y": -0.01, "z": 9.81, "ts": 1739500005}

// hydro-limoilou/poste-01/status
{"uptime": 3600, "rssi": -67, "link": "lora", "battery_v": 3.92, "ts": 1739500030}

// hydro-limoilou/poste-01/alarm/vibration
{"level": "warning", "value": 1.45, "unit": "m/s2", "ts": 1739500045}

7. Checkpoints d'évaluation

Le projet est évalué en 3 checkpoints à pondération croissante.

CP1 — Semaine 13 (5%)

Focus : intégration matérielle + 1ère publication MQTT + amorce KiCad

  • Critère 1.1 — Câblage breadboard conforme à l'assignation (1%, C1)
  • Critère 1.2 — Première publication MQTT vers le broker local (3%, C2)
  • Critère 1.3 — Amorce du shield KiCad (1%, C1)

CP2 — Semaine 14 (10%)

Focus : intégration logicielle complète + visibilité serveur central VM

  • Critère 2.1 — Tous les capteurs publient sur les bons topics (3%, C2)
  • Critère 2.2 — Alarmes fonctionnelles (2%, C1)
  • Critère 2.3 — Extension de l'interface tactile (3 pages) (1%, C1)
  • Critère 2.4 — Visibilité du site depuis le serveur central (3%, C2)
  • Critère 2.5 — Schéma KiCad complet (ERC sans erreurs) (1%, C1)

CP3 — Semaine 15 (20%)

Focus : démo 3 scénarios + finalisation PCB + livrables

  • Critère 3.1 — Démonstration des 3 scénarios (7%, C2)
  • Critère 3.2 — PCB finalisé (DRC + Gerbers + BOM) (3%, C1)
  • Critère 3.3 — Documentation processus de conception (1%, C1)
  • Critère 3.4 — Documentation protocoles et topics (3%, C2)
  • Critère 3.5 — Site visible côté VM pendant toute la démo (3%, C2)
  • Critère 3.6 — QoS et fiabilité MQTT (3%, C2)
Re-vérification au CP3 : à la démo finale, l'enseignant revérifie le bon fonctionnement des éléments validés aux CP1 et CP2, mais les notes des CP1 et CP2 ne sont pas révisées. CP1 + CP2 sont des prérequis fonctionnels au CP3.

8. Livrables attendus (à la fin de la sem. 15)

  1. Shield PCB (livrable de conception KiCad, non fabriqué) : schéma + PCB 2 couches + Gerbers + BOM
  2. Firmware (Arduino/ESP32, dans Git) : lecture capteurs, publication MQTT, alarmes
  3. Extension de l'interface tactile (Python sur Pi 5) : 3 pages (télémétrie, alarmes, état lien)
  4. Documentation (Markdown dans le dépôt) : description du site, schéma de câblage, topics utilisés, procédure de démo
  5. Vidéo de démo (≤ 5 min) : 3 scénarios (nominal, alarme, perte/reprise de lien)
